# 데이터 입력 가이드 — 비스트리그

> 데이터 입력 담당자를 위한 운영 매뉴얼  
> 마지막 수정: 2026-03-23

---

## 입력 원칙

**절대 지켜야 할 규칙:**

1. **선수 이름을 절대 포함하지 않는다** — 이니셜, 등번호, 닉네임도 불가
2. **타순 그룹 합산만 기록한다** — 개별 선수 성적이 아닌 그룹 합계
3. **구단 공식 명칭/로고를 사용하지 않는다** — 연고지 약칭만 사용

---

## 팀 코드 표

| 연고지 약칭 | KBO 코드 | 팀명 (참고용, 파일에 쓰지 않음) |
|-------------|----------|-------------------------------|
| 광주 | HT | KIA 타이거즈 |
| 대구 | SS | 삼성 라이온즈 |
| 서울L | LG | LG 트윈스 |
| 서울D | OB | 두산 베어스 |
| 수원 | KT | KT 위즈 |
| 인천 | SK | SSG 랜더스 |
| 대전 | HH | 한화 이글스 |
| 부산 | LT | 롯데 자이언츠 |
| 창원 | NC | NC 다이노스 |
| 서울K | WO | 키움 히어로즈 |

---

## 타순 그룹 정의

| groupType | 대상 타자 |
|-----------|----------|
| `leadoff` | 1번, 2번 타자 합산 |
| `cleanup` | 3번, 4번, 5번 타자 합산 |
| `lower` | 6번, 7번, 8번, 9번 타자 합산 |
| `starter_pitcher` | 선발투수 1명 (별도 입력) |

> 교체 타자가 해당 타순에 들어오면 그 타석도 포함해서 합산합니다.

## 입력 도구 2: quickInput (CSV 일괄 입력)

경기 수가 많거나 스프레드시트에서 복붙할 때 추천.

```bash
cd collector
npx ts-node src/tools/quickInput.ts --date 2026-04-01 --file data/input.csv
```

### CSV 형식

```
homeTeam,awayTeam,homeScore,awayScore,group,team,AB,H,2B,3B,HR,RBI,RUN,SB,BB,K
광주,대구,5,3,leadoff,광주,8,3,1,0,1,3,2,1,2,1
광주,대구,5,3,cleanup,광주,11,4,1,0,2,4,3,0,1,1
광주,대구,5,3,lower,광주,10,2,0,0,0,1,0,0,1,3
광주,대구,5,3,leadoff,대구,9,2,0,0,0,1,1,0,1,2
광주,대구,5,3,cleanup,대구,11,3,1,0,1,2,2,0,0,2
광주,대구,5,3,lower,대구,9,1,0,0,0,0,0,0,1,3
```

### 선발투수 행 추가 (선택)

`group=starter_pitcher`일 때 컬럼 재활용:

| CSV 컬럼 | 실제 의미 |
|----------|----------|
| AB | IP (투구 이닝) |
| H | PITCH (투구 수) |
| 2B | H (피안타) |
| 3B | K (탈삼진) |
| HR | BB (볼넷) |
| RBI | ER (자책점) |

```
광주,대구,5,3,starter_pitcher,광주,6.2,98,6,5,2,2,0,0,0,0
```

## 입력 도구 3: validate (검증)

저장 후 파일 이상 여부를 확인할 때 사용.

```bash
cd collector
npx ts-node src/tools/validate.ts --date 2026-04-01
```

**출력 예시:**

```
[✓] 20260401SSHT0.json — 정상
[✓] 20260401LGOB0.json — 정상
[✗] 20260401KTSK0.json — 에러:
    - batterGroups[0].stats: 단타가 음수가 됨 (H=1 < 2B+3B+HR=2)
요약: 3개 중 2개 정상, 1개 오류
```

---

## 자주 하는 실수와 해결법

### 경기 취소/우천 중단
- status를 `cancelled` 또는 `postponed`로 설정
- batterGroups는 빈 배열 대신 **실제로 진행된 이닝까지의 합산** 입력
- 점수와 스탯 모두 중단 시점 기준

### 더블헤더
- 두 경기를 별도 파일로 저장
- gameId 마지막 자리로 구분: `...0` (1차전), `...1` (2차전)
- quickInput에서는 같은 homeTeam+awayTeam 조합이 두 행 블록으로 중복 → 현재 자동 처리 미지원, inputHelper 사용 권장

### 대타/교체 처리
- 6번 타순에 대타가 들어왔어도 해당 타석은 `lower` 그룹에 합산
- 투수 교체 → 선발투수만 기록, 중간/마무리 투수는 기록하지 않음

### H < 2B + 3B + HR 에러
- 단타 계산 시 음수가 발생하는 경우
- 원인: H 수를 잘못 입력했거나, 안타 없이 장타만 체크한 경우
- 해결: H는 **2루타+3루타+홈런+단타의 합계** (모든 안타 합산)
